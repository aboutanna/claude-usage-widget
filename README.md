# Claude Usage Widget

macOS 悬浮小窗，实时显示 claude.ai 的用量进度条（5 小时会话 / 7 天各模型），以及 5 小时会话重置倒计时、7 天全部模型重置倒计时（两行分开显示）。菜单栏（非 Dock）常驻一个图标，可随时显示/隐藏/刷新/重新登录。

悬浮窗贴着屏幕左边或右边时，过一小会儿会自动收起成一个贴边细条，鼠标移上去就会展开；点击标题栏的 📌 按钮可以固定住，固定后不再自动收起，再点一下取消固定。

这是个人使用的非官方工具，原理和 Claude Usage Tracker、ClaudeKarma 等社区 Chrome 扩展完全一致：用你自己登录 claude.ai 后的 session cookie，调用 claude.ai 页面本身也会调用的用量接口 `GET /api/organizations/{orgId}/usage`，不涉及破解或绕过登录。

## 环境要求 

- macOS
- Node.js 18+（`node -v` 确认）

## 使用步骤

```bash
cd claude-usage-widget
npm install
npm start
```

首次启动：

1. 程序会弹出一个「Claude 登录」窗口（本质就是 claude.ai 网页），像平时一样登录你的账号。
2. 登录成功后，只要能正常打开设置页，登录窗口会在下一次轮询（最多 60 秒）后自动隐藏，悬浮小窗开始显示数据。
3. 之后每次启动 `npm start` 都会复用已保存的登录态，不需要重复登录（cookie 持久化在 Electron 的 `persist:claude-usage-widget` 分区里）。

悬浮小窗操作：

- 顶部标题栏可直接拖动到你想要的屏幕位置，位置会被记住；默认就贴在右边缘，所以刚启动没多久就会自动收起。
- 贴左/右边缘且未固定时，鼠标移开窗口约 0.5 秒后会自动收起成一根贴边细条（颜色代表当前最紧张的那项用量），鼠标移回细条上会自动展开。拖到屏幕中间的悬浮窗不会自动收起。
- `📌` 固定按钮：点亮后悬浮窗保持展开、不再自动收起；再点一次取消固定，恢复贴边自动收起的行为。固定状态会被记住，下次启动仍然有效。
- `⟳` 按钮立即刷新一次；`×` 按钮只是隐藏窗口，不会退出程序。
- 菜单栏图标（右上角一个圆形小图标）里可以：显示/隐藏悬浮窗、立即刷新、重新登录、退出程序。

## 打包成 .app / .dmg

```bash
npm run build
```

产物在 `dist/` 目录下（`.dmg` 和 `.zip`），应用图标使用 `assets/icon.icns`（一个暖橙色圆环图标，象征用量进度）。因为没有做 Apple 开发者签名，首次打开时 macOS 可能提示"应用已损坏"或"无法验证开发者"，属于正常现象，处理方式二选一：

- 右键点击 App → 打开，在弹窗里再次确认"打开"；或
- 终端执行：`xattr -cr "/Applications/Claude Usage Widget.app"`

## 已知限制

- 用量接口是 claude.ai 前端自己在用的内部接口，不是公开稳定的官方 API，Anthropic 若调整接口路径或响应结构，本工具会失效，需要相应修改 `main.js` 里 `fetchUsageFromSessionWindow` 的解析逻辑。
- 展示的字段（`five_hour` 会话、`seven_day` 系列周限额）取决于你的账号计划实际返回了哪些字段，不存在的字段会自动跳过不显示。
- 本项目未做代码签名/公证，仅建议本机个人使用，不建议分发给他人。

## 目录结构

```
main.js        Electron 主进程：隐藏的登录/抓取窗口、悬浮窗、菜单栏、轮询逻辑
preload.js     安全暴露给悬浮窗渲染进程的 IPC 接口
widget.html/.css/.js   悬浮小窗界面
assets/        菜单栏图标 + 应用图标（icon.icns）
```
